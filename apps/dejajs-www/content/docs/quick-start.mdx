# Quick Start

This guide assumes you have completed the [Installation](/docs/installation) steps. You will start the development servers, connect to a layout, and try the core operations: controlling a locomotive, throwing a turnout, and triggering an effect.

## Start the Development Servers

### All Apps at Once

From the repository root, run:

```bash
pnpm dev
```

This launches every app in the monorepo simultaneously using Turborepo. You will see output from each app in your terminal.

### Individual Apps

If you only need specific apps, use the `--filter` flag with the app's package name:

```bash
# Throttle app only
pnpm --filter=deja-throttle dev

# Cloud management app only
pnpm --filter=deja-cloud dev

# Server only
pnpm --filter=deja-serverts dev

# Server + Monitor together
pnpm deja
```

### App Ports

Each app runs on a fixed port in development:

| App | URL | Port |
|-----|-----|------|
| **Cloud** | `http://localhost:3011` | 3011 |
| **Tour** | `http://localhost:3021` | 3021 |
| **Monitor** | `http://localhost:3031` | 3031 |
| **Throttle** | `http://localhost:3041` | 3041 |
| **WebSocket Server** | `ws://localhost:8082` | 8082 |

The Sound API (Next.js) runs on port 3001 when started separately.

## Connect to a Layout

When you first open the Throttle or Cloud app, you will be prompted to select a layout.

![Select layout screen](/images/docs/quick-start/select-layout.png)

1. Open the Throttle app at `http://localhost:3041`.
2. Sign in with your Firebase credentials (or create an account if this is your first time).
3. Select your layout from the list, or create a new one.

Your `LAYOUT_ID` environment variable determines which layout the server connects to. The layout ID ties all data together -- locomotives, turnouts, effects, and commands are scoped to this ID in Firebase.

## Control a Locomotive

Once connected to a layout, you can drive a train from the Throttle app.

### Add a Locomotive to the Roster

Before you can control a locomotive, it needs to be in your roster. Open the Cloud app (`http://localhost:3011`) and navigate to the Roster section:

1. Click **Roster** in the navigation.
2. Add a new locomotive with its DCC address (the address programmed into the decoder).
3. Give it a name and optionally assign a photo.

![Add locomotive to roster](/images/docs/quick-start/add-loco.png)

### Drive a Train

Back in the Throttle app:

1. Select the locomotive from your roster.
2. Use the **speed slider** to set speed (0--126).
3. Tap the **direction button** to toggle between forward and reverse.
4. Use the **function buttons** (F0--F28) to control headlights, sounds, and other decoder functions.

![Throttle controls](/images/docs/quick-start/throttle-controls.png)

Behind the scenes, speed changes are written to Firebase Firestore at `layouts/{layoutId}/throttles/{address}`. The server picks up the change and sends a DCC-EX throttle command (`<t address speed direction>`) to the CommandStation over USB serial.

### Emergency Stop

The **Emergency Stop** button (visible on the throttle screen) immediately sets all locomotive speeds to zero. Use it any time you need to stop everything quickly.

## Throw a Turnout

Turnouts (track switches) are managed in the Cloud app and controlled from either the Throttle or Cloud app.

### Configure a Turnout

In the Cloud app:

1. Navigate to **Turnouts**.
2. Add a new turnout with its DCC-EX turnout index (the ID configured on your CommandStation).
3. Give it a name that describes its location (for example, "Yard Lead" or "Main/Siding Switch").

### Operate a Turnout

In the Throttle app, go to the **Turnouts** view:

1. You will see all configured turnouts with their current state (thrown or closed).
2. Tap a turnout to toggle its state.

![Turnout controls](/images/docs/quick-start/turnout-controls.png)

The turnout state is stored in Firestore at `layouts/{layoutId}/turnouts/{id}`. The server translates it into a DCC-EX command (`<T index state>`) and sends it to the CommandStation.

## Trigger an Effect

Effects let you control sounds, lights, animations, and other accessories on your layout.

### Configure an Effect

In the Cloud app, navigate to **Effects** and create a new effect. Effects can be:

- **Sound effects** -- Play audio files through speakers on the layout.
- **Light effects** -- Control LEDs and lighting scenes.
- **Accessory outputs** -- Toggle DCC accessory decoder outputs.

### Activate an Effect

In the Throttle app, go to the **Effects** view:

1. Browse the list of configured effects.
2. Tap an effect to activate it.

![Effects controls](/images/docs/quick-start/effects-controls.png)

## Useful Commands

Here are common commands you will use during development:

```bash
# Start all apps
pnpm dev

# Build everything (useful before committing)
pnpm build

# Type-check all packages
pnpm check-types

# Lint all packages (with auto-fix)
pnpm lint

# Format code
pnpm format

# Run throttle unit tests
pnpm --filter=deja-throttle test:unit
```
