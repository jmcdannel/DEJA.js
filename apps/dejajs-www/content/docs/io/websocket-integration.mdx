# рџЊђ WebSocket Integration

The DEJA.js server exposes a **WebSocket interface** for real-time bidirectional communication. This is the same protocol used by the Monitor, Throttle, and Cloud apps вЂ” so your custom scripts get the same live data stream as the official DEJA.js frontends.

Use WebSocket integration when you want to:
- рџ–ҐпёЏ Build custom dashboards or control panels
- рџ“Љ Log DCC commands and device data to a file or database
- рџ¤– Write automation scripts that react to layout events in real time
- рџ”Њ Bridge DEJA.js with other home automation or IoT platforms

## рџ”— Connection

```
ws://host:8082
```

| Parameter | Default | Env Variable |
|-----------|---------|--------------|
| Host | `localhost` | вЂ” |

If the server is behind a **Cloudflare Tunnel** or served over HTTPS, use `wss://` instead:


> рџ’Ў The WebSocket server must be enabled on the DEJA server with `ENABLE_WS=true` (this is the default).

## рџ¤ќ Connection Handshake

When you connect, the server sends two messages automatically вЂ” no request needed:

### 1пёЏвѓЈ `ack` вЂ” Connection Acknowledgment

Sent immediately on connect. Confirms the server identity and which layout it's managing.

```json
{
  "action": "ack",
  "payload": {
    "layoutId": "tamarack",
    "serverId": "DEJA.js"
  }
}
```

### 2пёЏвѓЈ `wsconnected` вЂ” Client Identification

Sent right after `ack`. Contains your client's IP address as seen by the server.

```json
{
  "action": "wsconnected",
  "payload": {
    "ip": "192.168.1.42",
    "serverId": "DEJA.js"
  }
}
```

Once you receive both messages, the connection is fully established and you can send commands or listen for broadcasts.

## рџ“¤ Sending Commands

Send JSON messages to the server with an `action` and `payload`:

```json
{
  "action": "string",
  "payload": {}
}
```

### Common Client-to-Server Actions

| Action | Payload | Description |
|--------|---------|-------------|
| `listPorts` | `{}` | Request a list of available serial ports |
| `getStatus` | `{}` | Request current server status |
| `ping` | `{}` | Ping the server (returns `status`) |
| `subscribe-device` | `{ "deviceId": "device-name" }` | Subscribe to serial data for a specific device |
| `unsubscribe-device` | `{ "deviceId": "device-name" }` | Unsubscribe from a device's serial data |

## рџ“Ґ Broadcast Messages from Server

These messages are broadcast to **all** connected WebSocket clients:

### `dcc` вЂ” DCC Command Log

Every DCC-EX command sent to the serial port is broadcast. Great for building a live command monitor.

```json
{
  "action": "dcc",
  "payload": "t 3 50 1"
}
```

### `connected` вЂ” Serial Port Connected

Broadcast when the server establishes a serial connection to a DCC-EX CommandStation.

```json
{
  "action": "connected",
  "payload": {
    "baudRate": 115200,
    "device": "CommandStation",
    "path": "/dev/tty.usbmodem1101"
  }
}
```

### `portList` вЂ” Available Serial Ports

Broadcast in response to a `listPorts` command.

```json
{
  "action": "portList",
  "payload": [
    "/dev/tty.usbmodem1101",
    "/dev/tty.usbserial-110"
  ]
}
```

### `status` вЂ” Server Status

Broadcast in response to `getStatus`, `status`, or `ping`.

```json
{
  "action": "status",
  "payload": {
    "client": "dejaJS",
    "isConnected": true
  }
}
```

### `serial-data` вЂ” Device Serial I/O

Sent **only** to clients subscribed to a specific device (see below).

```json
{
  "action": "serial-data",
  "payload": {
    "deviceId": "tj-eagle-nest-pico",
    "data": "<p1>",
    "timestamp": "2025-01-15T14:30:22.123Z",
    "direction": "incoming"
  }
}
```

The `direction` field is either `"incoming"` (device в†’ server) or `"outgoing"` (server в†’ device).

## рџ”Ќ Device Serial Monitoring

You can subscribe to a specific hardware device's serial I/O stream. This is useful for debugging, logging, or building device-specific dashboards.

### Subscribe to a Device

```json
{
  "action": "subscribe-device",
  "deviceId": "tj-eagle-nest-pico"
}
```

вњ… **Server response:**

```json
{
  "action": "device-subscribed",
  "payload": {
    "deviceId": "tj-eagle-nest-pico",
    "success": true
  }
}
```

### Unsubscribe from a Device

```json
{
  "action": "unsubscribe-device",
  "deviceId": "tj-eagle-nest-pico"
}
```

вњ… **Server response:**

```json
{
  "action": "device-unsubscribed",
  "payload": {
    "deviceId": "tj-eagle-nest-pico",
    "success": true
  }
}
```

> рџ’Ў You can subscribe to **multiple devices** simultaneously. When you disconnect, the server automatically cleans up all your subscriptions.

## рџљЂ Example: JavaScript WebSocket Client

Here's a complete example that connects to the DEJA server, listens for DCC commands, and subscribes to a device's serial stream.

### Browser

```javascript
const ws = new WebSocket('ws://localhost:8082')

ws.onopen = () => {
  console.log('рџ”— Connected to DEJA server')

  // Subscribe to a device's serial data
  ws.send(JSON.stringify({
    action: 'subscribe-device',
    deviceId: 'tj-thunder-city-pico'
  }))
}

ws.onmessage = (event) => {
  const msg = JSON.parse(event.data)

  switch (msg.action) {
    case 'ack':
      console.log(`рџ¤ќ Server ack вЂ” layout: ${msg.payload.layoutId}`)
      break

    case 'wsconnected':
      console.log(`рџЊђ Connected as ${msg.payload.ip}`)
      break

    case 'dcc':
      console.log(`рџљ‚ DCC command: ${msg.payload}`)
      break

    case 'device-subscribed':
      console.log(`рџ“Ў Subscribed to ${msg.payload.deviceId}`)
      break

    case 'serial-data':
      const { deviceId, data, direction } = msg.payload
      const arrow = direction === 'incoming' ? 'в¬…пёЏ' : 'вћЎпёЏ'
      console.log(`${arrow} ${deviceId}: ${data}`)
      break

    case 'connected':
      console.log(`рџ”Њ Serial connected: ${msg.payload.path}`)
      break

    case 'status':
      console.log(`рџ“Љ Status: connected=${msg.payload.isConnected}`)
      break

    default:
      console.log(`рџ“Ё ${msg.action}:`, msg.payload)
  }
}

ws.onclose = () => console.log('вќЊ Disconnected from DEJA server')
ws.onerror = (err) => console.error('вљ пёЏ WebSocket error:', err)
```

```javascript

```

## рџ§Є Testing Your Integration

1. **Start the DEJA server** вЂ” Make sure `ENABLE_WS=true` (default)
2. **Open a WebSocket connection** вЂ” Use the examples above or a tool like [websocat](https://github.com/vi/websocat):
   ```bash
   websocat ws://localhost:8082
   ```
3. **Watch the handshake** вЂ” You should immediately see `ack` and `wsconnected` messages
4. **Send a ping** вЂ” Type `{"action":"ping","payload":{}}` and confirm you get a `status` response
5. **Subscribe to a device** вЂ” Send a `subscribe-device` message and watch for `serial-data` events
