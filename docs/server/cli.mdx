# CLI Reference

The `deja` CLI is how you manage your DEJA Server from the terminal. It's installed automatically when you run the [installer](/docs/server/installation).

## Server Commands

### `deja start`

Start the server. By default, the server runs in the foreground with an interactive display.

| Flag | Description |
|------|-------------|
| `-b`, `--background`, `--bg` | Start in the background (no terminal window needed) |

**Interactive mode** (default) shows the DEJA.js banner and keeps the terminal attached. Type `exit` or press Ctrl+C to stop.

**Background mode** (`deja start -b`) runs the server silently. Use `deja status` to check on it and `deja stop` to shut it down.

### `deja stop`

Stop a running server. The server shuts down gracefully — it closes connections, disconnects from Firebase, and releases the serial port before exiting.

### `deja restart`

Stop the server and start it again. Accepts the same flags as `deja start` (e.g., `deja restart -b` to restart in background mode).

### `deja status`

Show the current state of your server, connections, and subscription. The output includes:

- **Server state** — running or stopped, version number, process ID
- **Account** — your User ID and Layout ID
- **Connections** — WebSocket port, MQTT status, Firebase Cloud status, detected serial ports
- **Remote monitoring** — tunnel status and URL (if applicable)
- **Apps** — links to Throttle, Cloud, and Monitor
- **Log file** — path and size of the current log file

### `deja logs`

View recent server log output.

| Flag | Description |
|------|-------------|
| `-f`, `--follow` | Follow logs in real time (like a live feed). Press Ctrl+C to stop. |
| `-n LINES`, `--lines LINES` | Number of lines to show (default: 50) |

**Examples:**

```bash
deja logs           # Show last 50 lines
deja logs -f        # Follow logs in real time
deja logs -n 100    # Show last 100 lines
deja logs -f -n 20  # Follow, starting from last 20 lines
```

### `deja update`

Check for and install the latest version of the DEJA Server. If the server is running, it will be stopped during the update and restarted automatically afterward.

The update preserves your configuration — your account, layout, and environment settings stay exactly as they were.

### `deja --version`

Print the installed server version.

### `deja help`

Show the full list of available commands.

## Tunnel Commands

Remote access lets you reach your [Monitor dashboard](https://monitor.dejajs.com) from outside your home network. It requires an Engineer or Conductor plan and the `cloudflared` tool. See [Remote Access](/docs/server/remote-access) for setup details.

### `deja tunnel start`

Start a secure Cloudflare tunnel. Once connected, the command prints a URL you can open from anywhere.

### `deja tunnel stop`

Stop a running tunnel.

### `deja tunnel status`

Check whether the tunnel is running and show the current URL.

### `deja tunnel logs`

View recent tunnel log output. Optionally pass a number to show more lines (e.g., `deja tunnel logs 100`).

## Understanding Console Output

When the server runs in the foreground (`deja start`), you'll see a live stream of status messages. Each line starts with a colored icon that tells you what kind of message it is:

### Message Types

| Icon | Level | Meaning |
|------|-------|---------|
| ▶ **start** | A service or process is launching |
| ✔ **success** | Something connected or completed successfully |
| ✖ **error** | Something went wrong (check the message for details) |
| ⚠ **warn** | A non-critical issue — the server keeps running |
| ℹ **info** | Informational — something happened, no action needed |
| ● **note** | Configuration detail or status update |
| ★ **star** | A DCC command was sent to the track |
| ✔ **complete** | A task finished (port opened, layout loaded, etc.) |
| … **await** | Waiting for something (serial write, reconnect, etc.) |
| ✖ **fatal** | A critical error — the server may need to be restarted |

### Startup Messages

When the server starts, you'll see messages like these in order:

1. **Running [MAIN]** — the server process has started
2. **Subscription valid** — your account was verified
3. **MQTT ON / WebSocket ON / DEJA Cloud ON** — which services are enabled
4. **Connecting to DejaCloud** — connecting to Firebase
5. **RTDB connection established** — real-time database is live
6. **Cleared stale entries** — cleaning up old command queues
7. **Throttles listening for loco changes** — ready to receive throttle commands
8. **Load layout / Layout loaded** — your layout configuration is loaded
9. **Auto connect device** — connecting to your CommandStation's serial port
10. **Port opened** — USB serial connection is active
11. **Connected to DejaCloud** — Firebase sync is fully established
12. **WebSocket server started** — browser apps can connect
13. **DEJA.js Server is running!** — everything is ready

### DCC Command Messages

When you control trains, turnouts, or effects, you'll see `★ star` messages confirming each command:

| Message | What happened |
|---------|---------------|
| **Throttle** `3 50 1` | Loco 3 set to speed 50, forward |
| **Turnout** `1 1` | Turnout 1 thrown |
| **Function** `3 0 1` | Loco 3 headlight (F0) turned on |
| **Power** `1` | Track power turned on |
| **Output** `{pin, state}` | An output pin was toggled |

### Shutdown Messages

When you stop the server (Ctrl+C or `/stop`), you'll see an orderly shutdown sequence:

1. **Graceful shutdown initiated** — shutdown process started
2. **Closing WebSocket server** — disconnecting browser clients
3. **Disconnecting from DEJA Cloud** — removing Firebase listeners
4. **Disconnecting MQTT client** — closing broker connection
5. **Stopping sound playback** — stopping any playing sounds
6. **Clearing audio cache** — cleaning up temporary files
7. **Disconnecting all serial ports** — releasing USB connections
8. **Shutdown complete** — clean exit

### Common Error Messages

| Message | What it means | What to do |
|---------|---------------|------------|
| **Subscription inactive** | Your subscription has expired | Visit [dejajs.com](https://dejajs.com) to renew |
| **Port is already in use** | Another server instance is running | Run `deja stop` first, or check for other processes on the port |
| **Error opening port** | Can't access the USB serial port | Check the cable, or run `sudo usermod -a -G dialout $USER` on Linux |
| **Broker not available** | MQTT broker isn't running | Start Mosquitto, or set `ENABLE_MQTT=false` in `~/.deja/.env` |
| **Firebase unreachable** | Can't connect to the cloud | Check your internet connection and Firebase credentials in `~/.deja/.env` |

For more solutions, see [Troubleshooting](/docs/server/troubleshooting).
