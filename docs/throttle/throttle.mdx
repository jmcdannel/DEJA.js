# Throttle Control

The Throttle Control screen (`/throttle/:address`) is where you drive a single locomotive. It provides everything you need for hands-on operation: speed adjustment, direction control, decoder function buttons, consist management, and an emergency stop.

## Screen Layout

The Throttle Control screen is divided into three main sections arranged in a row.

### Header

The header bar spans the full width of the screen and contains:

- **Locomotive avatar** -- a circular image representing the locomotive, sourced from the roster. Tapping the avatar opens a context menu with Park and Stop actions.
- **Mini consist indicator** -- if the locomotive is part of a consist (multiple units), small linked icons appear next to the avatar showing the other units in the group.
- **Locomotive name** -- displayed in a bold gradient text (red tones) next to the avatar.
- **Roadname logo** -- the railroad company logo (e.g., BNSF, Union Pacific, Norfolk Southern) appears on screens wider than 640px, pulled from the locomotive's metadata.
- **Park button** -- stops the locomotive (speed to zero) and releases the throttle, returning you to the Throttle List.

### Main Control Area

Below the header, the screen splits into up to three columns depending on screen width.

#### Left Column: Slider Controls (Desktop/Tablet Only)

Hidden on mobile (below 640px), the left column displays the **SliderControls** component with three sliders:

- **Speed slider** -- a horizontal Vuetify slider ranging from 0 to 126 (the DCC-EX speed range). Tick marks are drawn at each step, with labeled positions at intervals. The slider uses a 2-second debounce before sending the speed update to Firebase, preventing excessive writes during rapid adjustments.
- **Direction slider** -- a three-position switch: REV (reverse), IDLE (stopped), and FWD (forward). Direction can only be changed when the speed is at zero, enforcing safe operation.
- **Brake slider** -- a 0--10 range brake control for gradual deceleration.

Quick action buttons for the bell and headlights are also positioned alongside the sliders.

#### Center Column: Consist and Functions

The center column displays:

- **Consist component** -- shows the full consist (linked locomotive group) if the locomotive is part of one. Each member of the consist is displayed as a linked avatar.
- **Roadname logo** -- a larger version of the railroad logo, providing visual identity for the active locomotive.
- **Function speed dial** -- a floating action button that expands into a radial menu of decoder function buttons (F0 through F28). Common functions include:
  - F0: Headlight
  - F1: Bell
  - F2: Horn/Whistle
  - F3--F28: Decoder-specific features (dynamic brake, coupler, sounds, lighting effects)

Each function button shows an appropriate icon and toggles on/off when tapped.

#### Right Column: Button Controls and Speed Display

Always visible on all screen sizes, the right column contains:

- **Current speed display** -- a glowing pill-shaped badge showing the current signed speed value. The display uses an inset shadow with a purple-to-pink gradient background. Positive values indicate forward motion, negative values indicate reverse.
- **Button controls** -- five vertically stacked buttons for incremental speed adjustment:
  - **Double up** (chevron-double-up): Increase speed by 5 steps
  - **Up** (chevron-up): Increase speed by 1 step
  - **Stop** (stop icon): Immediately set speed to zero (red button, prominent center placement)
  - **Down** (chevron-down): Decrease speed by 1 step
  - **Double down** (chevron-double-down): Decrease speed by 5 steps

All buttons provide haptic feedback on supported devices -- a light vibration for speed adjustments and a heavy vibration for the stop button. Buttons use pointer capture for reliable touch interaction and display a green glow effect when pressed, or a red glow for the stop button.

## Swipe Navigation

On the Throttle Control screen, you can **swipe left or right** to switch between active throttles. The app detects swipe gestures using VueUse's `useSwipe` composable:

- **Swipe left** advances to the next throttle in your active throttle list.
- **Swipe right** returns to the previous throttle.

The list wraps around, so swiping past the last throttle returns to the first.

A **slide group** of locomotive avatar nav items appears at the bottom of the screen, showing all active throttles as tappable icons. Tapping any icon navigates directly to that throttle.

## Emergency Stop

The stop button on the Throttle Control screen sets the locomotive speed to zero immediately. For a layout-wide emergency stop, use the emergency stop button in the app header, which is visible on every page.

## Parking a Locomotive

Parking a locomotive performs two actions:

1. Sets the speed to zero.
2. Removes the throttle document from the Firestore collection at `layouts/{layoutId}/throttles/{address}`, releasing the throttle session.

After parking, you are redirected to the [Throttle List](/docs/throttle/throttle-list).

## No Throttle Assigned

If you navigate to `/throttle/:address` for an address that does not have an active throttle session, the screen displays a "No Throttle Assigned" message with a button to navigate to the Throttle List, where you can acquire a new throttle from the roster.

## How Speed Updates Work

Speed changes flow through the following path:

1. You press a button or adjust a slider on the Throttle Control screen.
2. The `useThrottle` composable writes the new speed and direction to Firestore at `layouts/{layoutId}/throttles/{address}`.
3. The DEJA.js server detects the Firestore change and translates it into a DCC-EX command: `<t address speed direction>`.
4. The command is sent over USB serial to the DCC-EX CommandStation.
5. The CommandStation adjusts the track signal, and the locomotive responds.

Speed values are signed internally: positive values mean forward, negative values mean reverse. The absolute value (0--126) represents the DCC speed step, and the sign determines the direction bit.
